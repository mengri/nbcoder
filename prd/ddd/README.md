# NBCoder DDD领域开发任务分解说明

本目录下为NBCoder系统按领域驱动设计（DDD）原则拆分的开发任务索引与说明。

- 每个领域均有独立的需求文档（*.md），详述领域职责、实体、功能。
- index.md 为全局任务索引，列出所有不可再分的开发任务，并明确任务依赖关系。
- 任务编号规则：领域首字母+序号（如R1.1、P1.2等），便于追踪与分工。
- 建议每个任务单独分支开发，完成后独立测试与集成。
