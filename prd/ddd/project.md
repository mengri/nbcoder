# 项目领域（Project Domain）需求说明

## 领域职责
- 负责项目元数据管理、多项目隔离、全局配置。
- 支持项目创建、配置、生命周期管理。

## 主要实体
- Project（项目）
- ProjectConfig（项目配置）
- Standards（开发规范）

## 主要功能需求
1. 支持多项目独立管理，目录与数据库隔离。
2. 支持项目创建、描述、Git 仓库配置。
3. 支持开发规范、分支策略、技术栈配置。
4. 支持全局与项目级配置分离。

## 限界上下文
- 仅管理项目元数据与配置，不直接管理需求卡片、任务、代码。
- 与 Requirement、Git 领域通过配置注入解耦。
