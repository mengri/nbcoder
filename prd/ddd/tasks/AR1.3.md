# 任务：AR1.3 Provider/模型可用性检测与降级

## 用户故事
作为一名平台运维者，我希望系统能自动检测Provider和模型的可用性，并在异常时自动降级，保障服务稳定。

## 领域上下文
- 所属领域：AI运行时领域（AI Runtime Domain）
- 领域职责：可用性检测、降级、与Agent集成。
- 相关聚合根/实体：Provider, Model, ModelStatus
- 依赖任务：AR1.1

## 任务目标
实现Provider/模型的可用性检测与自动降级机制。

## 输入
- 领域建模说明（见 prd/ddd/airuntime.md）
- 检测与降级规则

## 输出
- 可用性检测与降级实现代码

## 实现要点
- 支持多种检测与降级策略
- 检测与降级过程可追溯
- 与Agent/流水线集成

## 交付标准
- 代码可独立运行，单元测试覆盖主要检测与降级流程
- 检测与降级结构与领域建模一致
